# safe-whale — Specification

## Vision

`safe-whale` is a Tkinter GUI (and eventually a CLI) that lets you install and run Python packages (and eventually npm,
Ruby gems, etc.) inside Docker containers with a single click — as easy as `pipx`, but sandboxed. Think of it as a
desktop front-end to the concept already prototyped in `index.html`, but actually executable instead of generating
copy-paste text.

______________________________________________________________________

## Problem

Installing CLI tools from PyPI with `pipx` is convenient but trusting: the package runs with your full user context, can
read files, make network requests, exfiltrate data, etc. Docker isolation exists, but crafting the `docker run` or
`Dockerfile` correctly — with all the security knobs — is tedious enough that nobody does it. `safe-whale` makes the
secure path the easy path.

______________________________________________________________________

## Goals (MVP — Tkinter)

1. **Browse / search** a built-in catalog of popular Python CLI tools.
1. **Configure** the run with security knobs (non-root UID, read-only FS, cap-drop, memory/CPU/PID limits, network
   block, optional volume mount).
1. **Execute** — actually run the container (not just generate text), streaming output into the app.
1. **Save profiles** — persist a named configuration for a package so you can re-run it with one click.
1. **History** — log of past runs (package, args, timestamp, exit code).

______________________________________________________________________

## Non-Goals (MVP)

- No npm / Ruby / other ecosystems yet (architecture should leave a door open).
- No interactive TUI / shell-in-window in v1 (piping to a subprocess panel is acceptable; full PTY in tkinter is hard).
- No image registry browsing.
- No multi-container / compose.

## Data Model

```python
@dataclass
class CatalogEntry:
    name: str  # PyPI package name
    entrypoint: str  # CLI command name
    description: str
    apt_packages: list[str]  # system packages needed
    example_args: str
    ecosystem: str = "pypi"  # "pypi" | "npm" | "gem" (future)


@dataclass
class RunConfig:
    package_spec: str  # e.g. "httpie==3.2.2" or "yt-dlp[websockets]"
    entrypoint: str
    cli_args: str
    apt_packages: list[str]
    engine: str  # "docker" | "podman" | "nerdctl" | "finch"
    mode: str  # "run" (install-on-fly) | "dockerfile" (build first)
    interaction: str  # "immediate" | "interactive" | "pipe"
    # Security knobs
    read_only: bool = True
    no_new_privs: bool = True
    cap_drop_all: bool = True
    tmpfs_tmp: bool = True
    block_network: bool = False
    non_root: bool = True
    limit_pids: bool = True
    memory_mb: int = 1024
    cpus: float = 1.0
    mount_dir: str = ""  # "host_path:container_path[:rw|ro]"


@dataclass
class RunResult:
    config: RunConfig
    timestamp: datetime
    exit_code: int | None
    stdout: str
    stderr: str
```

## Container Command Builder (`container.py`)

### `run` mode (install-on-the-fly)

Builds a `docker run … python:3.13-slim sh -lc 'pip install … && exec <entry> <args>'` command, applying all security
flags from `RunConfig`.

### `dockerfile` mode

Generates a `Dockerfile` string (same logic as `index.html`), writes it to a temp dir, calls
`docker build -t safe-whale-<pkg>-<hash> .`, then runs with full hardening. The built image is cached so subsequent runs
are instant.

### Engine abstraction

All four engines (docker/podman/nerdctl/finch) use the same flag set — the builder just swaps the binary name.

## Security Design

All defaults follow the principle of least privilege:

- Non-root numeric UID (10001 / 65532)
- `--cap-drop=ALL`
- `--no-new-privileges`
- `--read-only` filesystem
- tmpfs for `/tmp` and `/var/tmp`
- Memory + CPU + PID limits
- No host filesystem mounts by default
- Optional `--network none` (disabled by default for packages that need PyPI at runtime)

The `Dockerfile` mode is the safer choice: install at build time, then run with `--network none`.

______________________________________________________________________

## Decisions Made

- **Engine detection**: Auto-detect docker/podman/nerdctl/finch at startup; default to first found; hide missing ones
  from the dropdown.
- **Output streaming**: Live streaming via background thread + `after()`. Also provide a "Run in Terminal" button that
  pops open a system terminal (Windows: `wt` / `cmd`, macOS: `Terminal.app`, Linux: `x-terminal-emulator`) running the
  command interactively.
- **Storage**: `platformdirs` user data directory (e.g. `%APPDATA%\safe-whale` on Windows, `~/.local/share/safe-whale` on
  Linux).
- **Runtime dependencies**: `platformdirs` only (besides stdlib + tkinter).
